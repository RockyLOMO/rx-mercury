校验 startTime <= endTime

## 前置通用流程
1. 是否需要先过 Sannysoft：是
2. 是否需要人工登录接管：是
3. 初始页最大等待时间：3分钟
4. 登录页最大等待时间：3分钟

## 页面操作流程
1. 打开https://pub.alimama.com/portal/v2/effect/order/overviewOrder/page/index.htm
2. 页面 付款时间下拉 右侧是 时间范围 选取组件（非原生的），

需要先点击下，然后下侧会弹出一个框， 框中 第一行是  选择时间：  字样，第二行就是 两个 日期选择 组件。

第二行任意点击日期选择，会再弹出一个框，要选择 任务入参的  startTime 日期， 和  endTime 日期。

假设 startTime =  2026-04-15，endTime = 2026-05-08
时间选取组件弹出框 顶部 < 是减少月份 选择上月  > 是增加月份 选择下月

startTime 选择逻辑：
直到 弹出框左侧顶部月份 是 startTime 月份部分 显示文字 2026年4月，
  然后 在弹出框左侧下部 点击一次 日期部分 数字15。

endTime 选择逻辑：
如果 弹出框左侧顶部月份 等于 endTime 月份部分， 那么直接在 弹出框左下侧 再点击一次 日期部分 数字。
如果 弹出框左侧顶部月份 不等于 endTime 月份部分，那么 弹出框右侧顶部 增加月份 直到 出框右侧顶部 文字 等于 endTime的月份部分。
然后在 弹出框右下侧 点击 日期部分数字。

3. 点击 搜索订单编号 placeholder 的 input 右侧的 查找订单按钮

5. 如果没有订单 返回 空数组。
如果有订单 按 任务出参部分 字段抓取成 数组。
把列表页的outerHtml解析抓取，或者你有更好的方式。
列表每行从左往右，从上往下，依次抓取
第1列中
  左边是图片忽略
  右边：
    第1行：productName + 超链接 productLink
    第2行：storeName
    第3行：mainOrderNo
    第4行：orderNo
    第5行：左边是estimatedBillingAmount 右边是orderTime
第2列：orderStatus
第3列：commissionRate
第4列：两行右侧的数字加起来 = estimatedCommission
第5列：两行右侧的数字加起来 = actualCommission  如果是 -- 则为null , 如果不为 --  actualBillingAmount = estimatedBillingAmount

6. 检查订单列表的 下一页 是否可点击，如果可以点击则点击，再抓取下一页的订单数据 直到 下一页不可点击。
然后把每一页的数据数据汇总返回即可。

## 验证用例
- 示例入参：近一个月的 startTime 和 endTime
- 期望结果：有多页订单数据返回
